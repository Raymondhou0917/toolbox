# 📝 更新紀錄（Changelog）

所有版本的更新紀錄都會記錄在這裡。格式參考 [Keep a Changelog](https://keepachangelog.com/)。

## [1.1.3] - 2024-12-09

### 新增
- **重置名單功能**：新增「重置名單」按鈕，讓用戶可以快速恢復預設名單，避免不小心刪除。只有預設名單組（雷蒙三十、健身鍻鍊、命運之輪）才會顯示此按鈕。

### 修正
- **輪盤動畫匹配問題**：修正多選項（10 個以上）時，指針指向與結果不匹配的問題。重構了旋轉角度計算邏輯，確保指針指向的扇形與顯示結果一致。

### 優化
- 動態調整輪盤字體大小：選項 > 8 個時使用 24px，> 6 個時使用 28px
- 增加旋轉圈數到 6 圈，動畫時間增加到 4 秒，讓動畫更流暢
- 改進指針設計，使用紅色內部和黑色外框，更明顯

## [1.1.1] - 2024-12-09

### 修正
- **輪盤動畫與結果匹配問題**：修正了指針指向位置與顯示結果不一致的 bug。現在輪盤停止時，指針指向的扇形會正確顯示為中獎結果。

### 新增
- **中獎後自動移除獲獎者**：在開始抽獎按鈕下方新增勾選框，勾選後中獎者會在 1.5 秒後自動從名單中移除。設定會自動儲存到 LocalStorage。

---

## [1.1.0] - 2024-12-09

### 變更
這個版本將專案簡化為**純前端靜態網站**，移除了後端依賴，讓部署更簡單。

### 移除
- 移除 tRPC、Express 後端
- 移除 MySQL 資料庫依賴
- 移除 Google OAuth 登入

### 保留
- 所有前端功能完整保留
- LocalStorage 本地暫存（重新整理不會消失）

### 變更
- 調整 Port 為 8080（Zeabur 預設）

### 為什麼這樣做？
原本的版本需要設定資料庫和 OAuth 環境變數，部署比較複雜。簡化版只需要一個靜態網站託管服務就能跑起來，更適合快速上線。

## [1.0.0] - 2024-12-09

### 首次發布 🎉

終於把工具箱整合起來了！原本抽獎工具和倒數計時器是分開的兩個專案，現在合併成一個，方便管理和部署。

### 新增
- **入口頁面**：Bento Grid 風格，可以選擇要用哪個工具
- **抽獎工具集**：輪盤、數字、分組三合一
- **名單組功能**：可以儲存常用名單，像是「雷蒙三十成員」、「健身鍛鍊項目」
- **本地暫存**：自動存在瀏覽器，重新整理不會消失
- **倒數計時器**：保留原本的設計，直接整合進來

### 技術決策
原本想用 Monorepo 把所有工具統一用 React 重寫，但發現倒數計時器重構後跑版嚴重。所以改用「方案 B」：把原本的 HTML/CSS/JS 直接放進 public 資料夾，完整保留原始設計。

這樣的好處是：
1. 不會跑版，原本長怎樣就怎樣
2. 維護簡單，改原本的 code 就好
3. 未來要加新工具，只要在入口頁加一張卡片就行
